Direction C · Hybrid · Foundation

The Condenser, on a 27" monitor.

The mobile design is the source of truth. The desktop version uses a left sidebar instead of a bottom tab bar, gives the high-density screens (Sites · Punch · Site detail) wider treatment, and keeps the simple screens (Today · Trades · Me · New punch) single-column. Below: the shell + the rules.

RULE 01

The breakpoint

min-width: 1024px — switch to sidebar. Below that, the bottom-tab-bar mobile layout stays exactly as-shipped. No tablet-specific layout for round 1; we treat tablet as "big phone."

RULE 02

The sidebar

Fixed 240px wide. Brand lockup top · orange New punch CTA · 4 nav items · profile chip + ⚙ at the bottom. Active item: ink background, paper text, orange 3px left bar. Inactive: muted icon + dark ink label.

RULE 03

The content area

Max-width 1240px, centered, with 32px gutters. Inside that, each screen picks its own column layout. Simple screens stay narrow (~760px max). Wide screens fill the canvas.

RULE 04

What stays the same

All tokens (color, type, spacing, radius, shadow). All atoms (Stamp · Pill · PriorityDot · stamped CTA). All copy. The only thing that changes is the layout container around the same React components.

Open questions before screen work begins
  • ⌘K command palette — I added the keyboard shortcut affordance but the actual search UX isn't designed yet. Want it for the desktop shell, or save for later?
  • Multiple workspaces — does a single account ever have more than one Condenser workspace (e.g., a CM working for two builders)? If yes, the sidebar top needs a workspace switcher.
  • Keyboard navigation — desktop = expected. The Site/Punch tables need arrow-key row selection, Enter to open, etc. Worth a spec doc on its own.
